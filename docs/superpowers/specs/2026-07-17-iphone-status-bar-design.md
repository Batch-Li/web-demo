# iPhone 16 状态栏重建设计

## 目标

将桌面演示机壳内的状态栏重建为可信的 iPhone 16 浅色状态栏。保持现有 393 × 852 活动屏幕、灵动岛、业务标题栏和移动端全屏逻辑不变。

## 现状问题

- 蜂窝信号、Wi-Fi 和电池被压缩在一个 67 × 14 SVG 中，尺寸和间距失真。
- 电池图形过小且始终满格，不像真实系统状态。
- 时间与右侧图标依赖通用 Flex 对齐，没有针对灵动岛保留左右安全区。
- 状态栏缺少可单独校准、测试和复用的结构。

## 设计方案

### 布局

- 状态栏固定高度 54px，不参与业务内容滚动。
- 时间位于左侧安全区，保留动态本地时间，使用系统字体和等宽数字。
- 右侧状态组拆为蜂窝信号、Wi-Fi、电池三个独立 SVG。
- 灵动岛维持 125 × 37px，状态内容不得侵入其水平安全区。
- 桌面机壳模式显示状态栏；宽度不超过 600px 的真实移动端继续隐藏模拟状态栏，交给浏览器/设备自身显示。

### 图标

- 蜂窝信号使用四级圆角柱，逐级增高。
- Wi-Fi 使用两条圆弧和中心圆点，以描边而非实心拼块绘制。
- 电池使用 25px 左右的外框、独立电池帽和 78% 填充量，填充与外框保持一致的黑色系统色。
- 三个图标拥有独立 class，允许分别校准尺寸和间距。

### 视觉约束

- 仅使用 CSS 和内联 SVG，不引入 Apple 官方受限素材或状态栏截图。
- 状态元素颜色使用 `currentColor`，浅色状态栏为近黑色。
- 不添加装饰动效；时间更新只反映真实分钟变化。

## 组件与数据

- `PhoneFrame` 继续负责机壳、状态栏、标题栏和 Home 指示条。
- 新增独立的状态图标结构，不引入第三方依赖。
- 时间在组件挂载时读取，并每分钟更新一次；卸载时清理定时器。

## 验收标准

- 393px 活动屏幕中，时间、灵动岛和右侧图标互不重叠。
- 电池图标可清晰辨认出外框、电池帽和非满格电量。
- 蜂窝、Wi-Fi、电池不再共享单个 SVG，独立尺寸和间距均可测试。
- 1280 × 900、1280 × 720 桌面视口下状态栏清晰；375px 移动视口不出现模拟状态栏。
- `npm test` 与 `npm run build` 通过，浏览器无控制台错误和横向溢出。
