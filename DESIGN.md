# 智绘适老 Web Demo 设计规范

## 0. 参考记录

- 视觉基础：沿用现有暖橙色适老界面，不改变扫描、商城、社区三入口结构。
- 设备外壳：参照 iPhone 16 的 393 x 852 pt 活动屏幕规格，以 CSS 和通用图标重建展示外壳，不使用受许可限制的 Apple 官方 Product Bezel 素材。
- 社区表达：参考真实生活方式社区的信息流，使用本地化真实照片、明确作者身份、相对时间和连续内容节奏。
- 商城表达：参考真实移动购物应用的商品数量、双列货架、横向补充货架和本地商品主图。

## 1. 产品气质

界面应当温暖、可信、清晰，面向老人及其家属。橙色只承担主行动和推荐强调，绿色承担安全状态，深色承担正文。展示页可以出现真实手机机身，但真实手机窄屏中不得再套设备边框。

## 2. 色彩与材质

- 主色：`#d86f2b`，深主色：`#8f4218`，浅主色：`#fff0df`。
- 安全色：`#4c725e`，浅安全色：`#eaf3ec`。
- 正文：`#211c18`，次级正文：`#73665b`。
- 页面底色：`#e9ece8`，内容面：`#f8f9f6`，强内容面：`#ffffff`。
- 设备外壳使用黑色 CSS 轮廓和通用状态图标，阴影仅用于设备与页面背景分离；App 内容面保持克制，不使用装饰性光斑。

## 3. 字体

使用苹方、微软雅黑、Noto Sans CJK SC 和系统字体回退。正文不小于 12px，关键操作不小于 14px，页面标题 22-28px。中文标题允许自然换行，但不得产生单字孤行或裁切。

## 4. 间距与圆角

- 基础间距为 4px，常用间距为 8、12、16、20、24px。
- App 内容组件默认圆角 8px；圆形操作和分段状态可使用全圆角。
- 手机外壳按统一的 409 x 868 展示画布和 393 x 852 活动屏幕实现；业务卡片不得复用设备圆角。

## 5. 可复用组件

- `PhoneFrame`：CSS 设备轮廓、393 x 852 pt 活动屏幕、54px 状态栏、灵动岛、App 标题栏和 Home 指示条。状态栏在灵动岛两侧使用固定安全区，时间、蜂窝、Wi-Fi 和 78% 电池图标均为独立结构。
- `BottomNav`：方案、扫描、效果社区三个固定入口，扫描居中强调。
- `CommerceTile`：商品图、店铺、名称、价格、规格、匹配原因和加入方案操作。
- `MallFeature`：评估后突出显示最高匹配商品。
- `MallRail`：展示双列货架之外的补充商品，必须可横向滚动。
- `CommunityFeedPost`：作者、时间、地点、单图/双图媒体、正文、复扫分数与互动操作；不同内容类型共享结构但不形成浮动卡片堆。
- 所有按钮必须具备默认、禁用、选中和键盘聚焦状态。

## 6. 图片

商品图必须与商品名称一一对应，优先使用白底或真实使用场景主图。社区图片必须来自真实住宅、老人生活或改造现场，不得使用生成式室内渲染。图片下载到本地并在 `public/assets/IMAGE_SOURCES.md` 登记来源，禁止依赖外部热链。商品图采用固定比例容器，社区图按内容采用 16:10 单图或 1:1 双图。

## 7. 动效与无障碍

动效仅用于扫描进度、分析状态和交互反馈，使用 `transform`、`opacity` 或 `filter`。所有图像提供语义化替代文本，控件保持 44px 左右触控面积，键盘聚焦轮廓清晰可见。

## 8. 响应式与已知边界

- 大于 600px：展示完整 iPhone 16 机身，活动屏幕按 393 x 852 pt 等比缩放。
- 小于等于 600px：隐藏机身、侧键、状态栏、灵动岛和 Home 指示条，App 全屏运行。
- Demo 中价格、评分和销量为演示数据，不代表外部商品页面的实时交易信息。
